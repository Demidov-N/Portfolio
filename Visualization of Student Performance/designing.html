Design Process

This sketch displays the strengths of the organization, #3 of our task analysis. This graph shows the average attendance per year of each school. Users can select specific schools they want to see. This graph addresses the task analysis by proving the increase in average attendance rate of most schools since the beginning, which means students enjoy their experience at BtC. We chose this graph because it easily shows the overall trend, and it includes user interaction in case the user wants to limit the options. Color encoding is included to separate each school. Vertical position is used to show data because average attendance is quantitative data. We used tableau to create the final sketch.


This sketch represents the distribution of attendance for each recommendation. The size of the circle shows the frequency. This sketch addresses our task analysis #2 : Did students improve their math skills after entering BtC? Since we were not offered any information about grades to address this problem, we concluded that students who are offered upper-class course recommendations are more likely to receive better grades. From this approach, we needed a design that can distinguish each recommendation and attendance rate. We implemented this design because it addresses all the points, and the user can easily understand the meaning of it.
